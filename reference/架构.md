## 1. 项目目标

本项目围绕长尾医学图像分类，当前实现分为两个阶段：

1. **Stage 1**：学习表征与原型记忆  
   使用 `encoder + classifier + projector + EMA teacher + prototype memory` 进行监督训练。
2. **Stage 2**：冻结 Stage 1 encoder 与 projector，只训练 classifier，在固定投影空间中对 hardest classes 生成、筛选、重建并再生成虚拟特征。

当前仓库的主线不是一个通用框架，而是一套围绕 ISIC2019LT / ISIC_Archive 的实验代码。运行入口、超参和产物路径主要由 `scripts/` 目录管理。

## 3. 整体流程

### 3.1 Stage 1

输入图像后，使用强弱双视图训练：

1. 从 `ISICDataset` 读取单张图像和类别标签
2. `PairTransformDataset` 生成：
   - `strong`：强增强视图
   - `weak`：弱增强视图
3. `encoder(strong)` 产生 student feature
4. `encoder_teacher(weak)` 产生 teacher feature
5. `classifier(student feature)` 做主分类
6. `projector(student feature)` 得到原型/对比空间特征 `z_s`
7. `prototype memory` 维护每类原型
8. 通过分类损失、PCD、SPKD、prototype separation 联合训练
9. 每个 batch 结束后更新：
   - EMA teacher
   - prototype memory

输出产物：

- `resnet_encoder_{best/latest}.pth`
- `classifier_{best/latest}.pth`
- `projector_{best/latest}.pth`
- `prototype_memory_{best/latest}.pth`
- `per_class_metrics.csv`
- loss / group curve 图片

### 3.2 Stage 2

Stage 2 当前主线是“冻结 encoder + projector，只训练 classifier”的闭环训练：

1. 加载 Stage 1 `encoder + projector + prototype`
2. 固定 encoder，预提取 train / val / test backbone feature
3. 用冻结的 projector 将真实特征投影到 Stage 2 空间
4. 在所有类别中按当前最难类选择 `hardest classes`
5. 对 hardest classes 生成首轮候选虚拟特征
6. 用当前 classifier + 类中心约束筛选可信虚拟特征
7. 用 `真实特征 + 通过筛选的虚拟特征` 重算 `mu / cov`
8. 再生成一轮虚拟特征，并再次筛选后参与训练
9. 用真实样本与虚拟样本共同训练 classifier
10. projector 保持冻结，不参与 Stage 2 优化
11. 评估并导出分析结果

输出产物：

- `classifier_{best/latest}.pth`
- `projector_{best/latest}.pth`（当前只是保存一份 Stage 2 运行时使用的冻结 projector）
- `gaussian_stats_history.csv`
- `per_class_metrics.csv`
- group curve 与 loss curve

## 4. 数据与分组逻辑

### 4.1 数据集读取

`data/dataset.py` 中的 `ISICDataset`：

- 输入：`root_dir + csv_file`
- CSV 第一列为 `image`
- 后续列为 one-hot 类别列
- 通过 `np.argmax` 转为单标签类别索引

图像路径规则：

- 优先读取 `image_name.jpg`
- 失败后尝试 `image_name.JPG`

### 4.2 训练/测试变换

`data/transforms.py`：

- **weak transform**：resize + flip + normalize
- **strong transform**：在 weak 基础上加入 blur、几何变换、颜色扰动、grid dropout
- **test transform**：确定性 resize + normalize

### 4.3 head / medium / tail 分组

分组实现见：

- `train_stage1.py::split_groups`
- `utils/metrics.py::build_group_specs`

规则是：

1. 统计训练集各类样本数
2. 按数量降序排序
3. `np.array_split(order, 3)` 均分成三组

这意味着：

- 分组是按**相对频次排序**划分
- 不是按固定样本阈值划分
- 每组类数由类别总数决定

## 5. Stage 1 模型架构

### 5.1 Backbone

实现：`models/encoders.py`

支持：

- `resnet18`
- `resnet34`
- `resnet50`
- `medclip_vit`

当前脚本主用：

- `backbone=medclip_vit`

`ResNetBackbone` 输出：

- `resnet18/resnet34`：`feat_dim=512`
- `resnet50`：`feat_dim=2048`
- `medclip_vit`：`feat_dim=512`

`medclip_vit` 当前支持两种离线初始化路径：

1. 通过 `MEDCLIP_VISION_MODEL_DIR` / `MEDCLIP_VIT_DIR` 指向本地 Hugging Face `swin-tiny-patch4-window7-224` 目录
2. 如果没有完整 HF 模型目录，则回退到仓库内的本地 fallback：
   - `pretrained/medclip-vit/config.json`
   - `pretrained/medclip-vit/pytorch_model.bin`

也就是说，当前仓库已经不再强依赖联网拉取 `microsoft/swin-tiny-patch4-window7-224`，只要本地 `config.json + MedCLIP 权重` 完整即可离线启动。

### 5.2 Projector

实现：`models/heads.py::Projector`

作用：

- 将 backbone feature 映射到原型空间
- Stage 1 中用于 PCD、SPKD、prototype memory
- Stage 2 中用于特征抽取与虚拟特征建模

支持：

- 单层线性投影
- 两层 MLP 投影（当 `proj_hidden_dim > 0`）

当前主用：

- `proj_dim=128`
- `proj_hidden_dim=0`

### 5.3 分类头

Stage 1：

- `nn.Linear(feat_dim, num_classes)`

Stage 2：

- `CosineClassifier(in_dim, num_classes, scale)`

### 5.4 EMA Teacher

Stage 1 中维护一个 teacher encoder（以及可选 projector teacher）：

- 初始化为 student 拷贝
- 每个 step 后用 EMA 更新

公式：

```text
theta_teacher <- ema_decay * theta_teacher + (1 - ema_decay) * theta_student
```

当前默认：

- `ema_decay=0.999`（训练入口默认）
- `config/configs.yaml` 中仍保留 `0.99`，与代码主线略有偏差

## 6. Stage 1 损失设计

Stage 1 总损失：

```text
L_stage1 = L_cls + lambda_pcd * L_pcd + lambda_spkd * L_spkd + L_proto_sep
```

其中 `L_proto_sep` 在日志和变量命名中仍沿用旧名 `punif` / `loss_punif`，但当前实现已经切换为**最近邻 prototype separation**，不再是旧的全局 uniformity。

### 6.1 分类损失 `L_cls`

支持三种模式：

- `ce`
- `balanced_softmax`
- `deferred_balanced_softmax`

当前脚本默认：

- `cls_loss=ce`

若启用 deferred 版本，则：

```text
epoch < cls_warmup_epochs: ordinary CE
epoch >= cls_warmup_epochs: balanced softmax
```

### 6.2 PCD：Prototype Compactness Distillation

实现：`utils/losses.py::pcd_loss`

定义：

```text
z_i = normalize(projected_feature_i)
p_y = normalize(prototype_of_class_y)
L_pcd = mean_i max(m - cos(z_i, p_yi), 0)
```

当前主用参数：

- `pcd_weight=1.0`
- `pcd_margin=0.85`

含义：

- 鼓励样本向所属类原型靠近
- 但使用 margin hinge，而不是直接最大化余弦相似度

### 6.3 SPKD：Similarity-Preserving KD

实现：`utils/losses.py::sp_kd_loss`

核心步骤：

1. 构造 student / teacher 与 memory queue 拼接后的 Gram 矩阵
2. 对 Gram 矩阵做 Frobenius norm 归一化
3. 最小化两者的平方差

公式可写为：

```text
G_s = Z_s Z_s_all^T
G_t = Z_t Z_t_all^T
L_spkd = || G_s / ||G_s||_F - G_t / ||G_t||_F ||_F^2
```

当前主用参数：

- `spkd_weight=10.0`

### 6.4 当前 prototype 分离项：最近邻 separation

实现：`utils/losses.py::prototype_nearest_neighbor_separation_loss`

当前实现不是旧的：

```text
logsumexp(all pair similarities)
```

而是：

```text
L_proto_sep = mean_i max(cos(p_i, p_nn(i)) - m_sep, 0)
```

其中：

- `p_i`：第 `i` 个类原型
- `p_nn(i)`：与 `p_i` 余弦相似度最高的其他类原型
- `m_sep`：分离 margin

当前新增参数：

- `proto_sep_margin`，默认 `0.5`

这个设计比全局 uniformity 更温和，只约束“最近的原型邻居”，更接近解决 medium 类混淆的目标。

### 6.5 该项的 warmup

虽然当前 regularizer 已经从 `uniformity` 切到 `nearest-neighbor separation`，但 warmup 逻辑仍沿用 `punif_*` 命名：

```text
w(epoch) = 0                           if epoch <= warmup_start
w(epoch) = linear ramp                if warmup_start < epoch <= warmup_end
w(epoch) = target_weight              afterwards
```

当前 Stage 1 代码会校验：

```text
punif_warmup_end_epoch >= punif_warmup_start_epoch
```

## 7. Prototype memory 与更新机制

### 7.1 batch class mean

每个 batch 中，对出现过的类别 `c` 计算：

```text
mu_c^(batch) = mean(z_i),  where y_i = c
```

### 7.2 EMA prototype 更新

每个 batch 的基础更新：

```text
p_c <- m * p_c + (1 - m) * mu_c^(batch)
```

当前主用：

- `proto_momentum=0.96`

### 7.3 recalibration

当满足：

```text
epoch % recal_interval == 0
```

时，再额外做一次：

```text
p_c <- (1 - alpha_c) * p_c + alpha_c * mu_c^(batch)
```

其中：

```text
alpha_c = recal_alpha * recal_factor_c
```

若启用 tail 补偿：

```text
recal_factor_c = 1.0           for head/medium
recal_factor_c = tail_factor   for tail
```

当前主线已经把 `tail_factor` 做成了可配置参数：

- `recal_tail_factor`

### 7.4 当前脚本默认

当前 `scripts/run_stage1_*.sh` 默认：

- `RECAL_TAIL_FACTOR=1.0`

也就是说当前脚本默认是**关闭 tail 额外加速**的。

- `val/test acc`
- `val/test bac`
- `val/test bacc`

### 9.2 训练中间产物

每个 run 目录下会保存：

- `train_loss_curve.png`
- `val_group_curves.png`
- `test_group_curves.png`
- `per_class_metrics.csv`

### 9.3 指标定义

#### 整体 acc

```text
overall_acc = 正确预测总数 / 总样本数
```

#### per-class acc（当前代码语义）

当前 `utils/metrics.py` 中：

```text
per_class_acc(c) = TP_c / (TP_c + FN_c)
```

也就是 **per-class recall**。  
字段名保留为 `acc` 是历史命名，不是 one-vs-rest accuracy。

#### group acc

当前 `head/medium/tail acc` 是组内宏平均：

```text
group_acc(S) = mean_{c in S} per_class_acc(c)
```

因此它本质上是：

```text
group macro recall
```

#### `bac` 与 `bacc`

当前代码里：

- `bac`：per-class recall
- `bacc`：`0.5 * (recall + specificity)`

整体层面的 `balanced_accuracy_score` 则由 `sklearn` 计算。

## 10. Stage 2 设计细节

### 10.1 输入

Stage 2 依赖 Stage 1 的：

- encoder checkpoint
- projector checkpoint
- prototype checkpoint

### 10.2 特征与统计量

当前 Stage 2 会先固定 encoder，并在训练开始时一次性提取：

- `train_backbone_feats`
- `val_backbone_feats`
- `test_backbone_feats`

然后使用**冻结的 Stage 1 projector**将这些 backbone feature 一次性映射到投影空间，得到：

- `train_proj_feats`
- `val_proj_feats`
- `test_proj_feats`

之后对 `train_proj_feats` 计算：

- 每类均值 `mu_c`
- 每类标准差 `sigma_c`
- 全局共享协方差 `shared_cov`

### 10.3 原型与均值融合

```text
mu_tilde = lambda_mu * mu + (1 - lambda_mu) * prototype
```

当前默认：

- `lambda_mu=0.5`

### 10.4 hardest-class 分配

先计算每类训练集准确率：

```text
acc_c = correct_c / total_c
```

再按：

```text
score_c = exp(alpha * (1 - acc_c))
```

但当前实现不是对全类平均分配，而是：

1. 先从所有类别中选出 hardest classes
2. 只在 hardest classes 内做 softmax 分配
3. 非 hardest 类的 `alloc_c = 0`

可控参数：

- `hardest_k`
- `hardest_fraction`
- `aas_alpha`

这样做的目标是把虚拟样本预算集中给当前最难的类别，而不是把预算摊给全部类别。

### 10.5 虚拟特征采样与筛选

当前主线使用共享协方差采样：

```text
z ~ N(mu_tilde_c, cov_scale_factor * shared_cov + delta^2 I)
```

但每个 epoch 的流程不是“采一次就直接训练”，而是：

1. 用真实特征统计量生成**首轮候选虚拟特征**
2. 通过筛选规则保留可信样本
3. 用 `真实特征 + 通过筛选的虚拟特征` 重算 `mu / cov`
4. 再生成**第二轮虚拟特征**
5. 第二轮虚拟特征也再做一次筛选后才用于训练

当前筛选规则包括：

- 预测类别必须等于目标类别
- softmax 置信度必须高于阈值
- 虚拟特征与所属类中心的 cosine 相似度必须高于阈值

对应参数：

- `virtual_conf_thresh`
- `virtual_center_cos_thresh`

当前实现**不跨 epoch 保存虚拟特征向量**。  
也就是说每个 epoch 都重新：

```text
生成 -> 筛选 -> 重算 mu/cov -> 再生成
```

上一轮虚拟特征不会直接缓存到下一轮使用。

### 10.6 冻结 projector、训练 classifier

当前 Stage 2 已重新收回到“固定 projector、只训练分类头”的版本。

当前训练策略是：

- **encoder 冻结**
- **projector 冻结**
- **classifier 参与训练**

其中：

- 真实样本使用预先计算好的 `projected feature -> classifier` 产生分类损失
- 虚拟样本已经位于投影空间内，因此只对 classifier 提供额外监督

这意味着：

- Stage 2 的投影空间在整个训练过程中保持不变
- classifier 同时接受真实样本和虚拟样本监督

### 10.7 anchor loss

当前代码中 `anchor loss` 不再参与优化。此前它用于约束可训练 projector 不要偏离 Stage 1 空间；在 projector 重新冻结后，这一项已经退回为兼容保留参数。

总训练目标可概括为：

```text
L_stage2 = L_real_cls + lambda_virtual * L_virtual_cls
```

对应参数：

- `virtual_loss_weight`

其中：

- `projector_lr`
- `anchor_weight`

当前仍保留在 CLI 中，但在 projector 冻结模式下不参与优化，只为兼容现有脚本与历史命令行。

### 10.8 分类头初始化

当前 `CosineClassifier` 在 Stage 2 初始化时，会直接用 Stage 1 prototype 作为初始权重方向。

这样做的目的有两个：

1. 避免 classifier 从完全随机状态开始
2. 让 hardest-class 判定在早期 epoch 更有意义
