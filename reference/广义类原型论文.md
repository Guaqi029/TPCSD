## 1. 传统原型方法的问题是什么？

在人 Re-ID 任务中，每张图片会先经过 Backbone，例如 ResNet50、TransReID、PHA，变成一个特征向量。

假设某个人有多张图像：

$$x_1, x_2, x_3, ..., x_s$$

经过 Backbone 后得到：

$$F(x_1), F(x_2), F(x_3), ..., F(x_s)$$

传统 centroid-based prototype 会直接取平均：

$$p = \frac{1}{s}\sum_{i=1}^{s}F(x_i)$$

也就是说，一个人只用一个中心点表示。

## 2. GCP 的目标是什么？

GCP 的目标是：对于一个类别 $c_i$，自动得到多个原型：

$$p^{c_i}_1, p^{c_i}_2, ..., p^{c_i}_N$$

查询图像 $q$ 来了以后，不再只和一个中心点比较，而是和该类的多个原型比较。

例如某个人有 3 个原型：

$$p_1, p_2, p_3$$

查询图像只要靠近其中任意一个原型，就可以认为它属于这个人。论文把这个目标写成类似下面的形式：

$$\exists k\in\{1,...,N_{c_i}\}, \forall c_j:c_j\neq c_i  
\|p^{c_i}_k - q^{c_i}\| < \|p^{c_j}_{k'} - q^{c_i}\|$$

意思是：对于真实类别 $c_i$ 的查询样本，至少存在一个该类原型，比其他类别的任何原型都更接近它。

## 3. 它怎么“自动得到多个原型”？

它不是简单地用 k-means 聚类，而是训练了一个 **Transformer Decoder-based model**。

整体流程是：

$$\text{同一类的多张图像} \rightarrow \text{Backbone提特征} \rightarrow \text{Transformer Decoder} \rightarrow \text{多个原型}$$

具体来说：

对于类别 $c$ 的多张图像：

$$x^c_1, x^c_2, ..., x^c_s$$

先用预训练 Backbone 提取特征：

$$F(x^c_1), F(x^c_2), ..., F(x^c_s)$$

这些特征不是直接平均，而是作为 Transformer Decoder 的 **memory**。

论文中写成：

$$M^c = [F(x^c_1), F(x^c_2), ..., F(x^c_s)] + \psi$$

其中 $\psi$ 是 camera ID 对应的位置嵌入，用来告诉模型这些图像来自哪个摄像头、姿态或视角信息。

## 6. Attention 机制具体起什么作用？

Decoder 里面有两种注意力：

### Self-Attention

看已经生成过的原型：

$$p_1, p_2, ..., p_{t-1}$$

作用是：让当前原型知道“前面几个原型已经代表了哪些区域”，避免重复。

### Cross-Attention

看该类别所有图像特征：

$$F(x^c_1), F(x^c_2), ..., F(x^c_s)$$

作用是：让当前原型根据整个类别的分布来生成，而不是只看某一张图片。

所以 GCP 的生成逻辑是：

$$p_t = Decoder([\langle sos\rangle, p_1,...,p_{t-1}], M^c)$$

其中 $M^c$ 是该类别所有图像特征的集合。

## 7. 它怎么训练这些原型？

训练目标有两个：

第一，同类样本要靠近本类原型。

第二，不同类样本要远离彼此的原型。

论文使用的损失函数是：

$$L = L_{triplet} + \lambda L_{reg}$$

其中：

$$L_{triplet} = max(0, m + \|a-p\|_2 - \|a-n\|_2)$$

这里 $a$ 是 anchor，$p$ 是 positive，$n$ 是 negative。也就是让同类距离小，不同类距离大。

Muzaddid和Beksi - 2026 - Person …

但如果只用 triplet loss，会出现一个问题：

**同一类的多个原型可能塌缩到同一个点。**

也就是：

$$p^c_1 \approx p^c_2 \approx p^c_3$$

这样就又退化成单原型了。

所以论文加了一个正则项：

$$L_{reg}$$

用于鼓励同一类别内的多个原型保持一定距离，避免全部重合。论文明确说，如果只使用 triplet loss，同一类别的原型会在特征空间中塌缩到一个点，因此引入 $L_{reg}$ 来防止塌缩并鼓励多样性。

## 8. 推理阶段怎么用这些原型？

推理时，流程是：

给定 gallery 中某个类别的所有图像：

$$x^c_1, x^c_2, ..., x^c_s$$

先提取特征，组成 memory：

$$M^c$$

然后 GCP 生成 $N$ 个原型：

$$p^c_1, p^c_2, ..., p^c_N$$

对于查询图像 $q$，先提取查询特征：

$$F(q)$$

然后和所有类别的所有原型算距离：

$$d(F(q), p^c_k)$$

最后选择最近的那个原型对应的类别。

也就是说，分类/检索规则变成：

$$\hat{c} = \arg\min_c \min_k \|F(q) - p^c_k\|$$

它不是问“查询图像离哪个类别中心最近”，而是问：

> 查询图像离哪个类别的某个代表点最近？

这样就允许一个类别有多个局部中心。

# 一、你现在的单原型逻辑是什么？

你当前 Stage 1 的核心是：

```
prototype_memory: [num_classes, proj_dim]
```

每个类只有一个原型：

$$p_c \in \mathbb{R}^D$$

每个 batch 中，对类别 $c$ 的样本特征求均值：

$$\mu_c^{batch} = mean(z_i), \quad y_i=c$$

然后 EMA 更新：

$$p_c \leftarrow m p_c + (1-m)\mu_c^{batch}$$

同时 PCD loss 让样本靠近自己类别的这个唯一原型：

$$L_{pcd} = mean_i \max(m - cos(z_i, p_{y_i}), 0)$$

这就会强迫一个类在投影空间里收缩到一个点。

# 二、最小改造目标

你要把：

```
每类 1 个原型
prototype_memory[c]
```

改成：

```
每类 K 个原型
prototype_memory[c, k]
```

也就是：

$$P_c = \{p_{c,1}, p_{c,2}, ..., p_{c,K}\}$$

对于样本 $z_i$，不再要求它靠近唯一的 $p_c$，而是靠近本类中最相似的那个原型：

$$p^*_{y_i} = \arg\max_{k} cos(z_i, p_{y_i,k})$$

$$L_{mpcd} = mean_i \max(m - \max_k cos(z_i, p_{y_i,k}), 0)$$

这样就实现了最核心的多原型思想：

> 一个类内部可以自然分成多个局部簇，每个样本只需要靠近自己最匹配的那个类内原型，而不是被迫靠近全类中心。

# 四、PCD loss 怎么改？

原来的 PCD 是：

```Python
proto = prototype_memory[labels]   # [B, D]
sim = (z * proto).sum(dim=1)
loss = relu(margin - sim).mean()
```

多原型版改成：

```Python
class_protos = prototype_memory[labels]      # [B, K, D]
z_expand = z.unsqueeze(1)                    # [B, 1, D]

sim = (z_expand * class_protos).sum(dim=-1)  # [B, K]
max_sim, assign_k = sim.max(dim=1)           # [B]

loss = F.relu(margin - max_sim).mean()
```

对应公式：

$$L_{mpcd} = \frac{1}{B}\sum_i \max(0, m - \max_k cos(z_i, p_{y_i,k}))$$

这一步是最关键的，改动也最小。

# 五、prototype memory 更新怎么改？

原来是每个类一个 batch mean：

$$p_c \leftarrow m p_c + (1-m)\mu_c^{batch}$$

现在改成：**每个样本先分配给本类最近的原型，然后每个原型只用分配给它的样本更新。**

流程如下：

```Python
for c in classes_in_batch:
    z_c = z[labels == c]                      # [Nc, D]
    protos_c = prototype_memory[c]            # [K, D]

    sim = z_c @ protos_c.T                    # [Nc, K]
    assign = sim.argmax(dim=1)                # [Nc]

    for k in range(K):
        z_ck = z_c[assign == k]
        if len(z_ck) > 0:
            mu_ck = z_ck.mean(dim=0)
            prototype_memory[c, k] = momentum * prototype_memory[c, k] \
                                   + (1 - momentum) * mu_ck
```

然后归一化：

```Python
prototype_memory[c, k] = F.normalize(prototype_memory[c, k], dim=0)
```

这就是一个 **online k-means + EMA** 的版本。

它不需要 Transformer，不需要重写模型结构，只是把已有 prototype memory 升级成多槽位 memory。

# 六、怎么防止一个类的多个原型塌缩？

这是必须加的。否则 $p_{c,1}, p_{c,2}, p_{c,3}$ 很可能又变成差不多的点。

你现在已经有一个“最近邻 prototype separation”项，用来分离不同类原型。

多原型之后，需要拆成两个分离项：

## 1. 类间分离：不同类别原型之间要远

$$L_{inter} = mean \max(cos(p_{c,k}, p_{c',k'}) - m_{inter}, 0)$$

其中：

$$c \neq c'$$

这相当于你当前 prototype nearest-neighbor separation 的多原型版本。

## 2. 类内多样性：同一类别内部的多个原型不要完全重合

$$L_{intra-div} = mean \max(cos(p_{c,k}, p_{c,k'}) - m_{intra}, 0)$$

其中：

$$k \neq k'$$

注意，这里的目标不是让同类原型非常远，而是**不要完全重合**。所以 margin 要设得宽松一些。

建议初始：

```
proto_inter_margin = 0.5
proto_intra_margin = 0.85 或 0.9
```

含义是：

* 不同类原型相似度超过 0.5 就惩罚；
* 同类不同原型相似度超过 0.85/0.9 才惩罚。

因为同一类的多个原型本来就应该有一定相似性，不能把它们推得太散。

# 八、Stage 2 怎么最小改？

你当前 Stage 2 会用 Stage 1 prototype 初始化 `CosineClassifier`，并且用：

$$\mu_{tilde} = \lambda_\mu \mu + (1-\lambda_\mu) prototype$$

来融合类均值和 prototype。

粘贴的 markdown (1)。md

多原型之后，Stage 2 有两种轻量改法。

---

## 方案 A：Stage 2 仍然使用单中心，取多原型平均

这是最小改动：

$$\bar{p}_c = \frac{1}{K_c}\sum_k p_{c,k}$$

然后原来的 Stage 2 不变：

$$\mu_{tilde,c} = \lambda_\mu \mu_c + (1-\lambda_\mu)\bar{p}_c$$

优点：改动极小。

缺点：又部分退回单中心思想。

---

## 方案 B：Stage 2 给每个类生成多个高斯中心

更符合你的目标。

原来每类一个：

$$z \sim \mathcal{N}(\mu_{tilde,c}, \Sigma)$$

改成每类多个：

$$z \sim \mathcal{N}(\mu_{tilde,c,k}, \Sigma)$$

其中：

$$\mu_{tilde,c,k} = \lambda_\mu \mu_{c,k} + (1-\lambda_\mu)p_{c,k}$$

$\mu_{c,k}$ 可以通过真实样本分配得到：

```
每个真实样本 z_i 分配给最近的 p_{c,k}
然后计算每个子簇均值 mu_{c,k}
```

然后每个 hardest class 的虚拟样本预算再分到各个原型簇：

```
alloc_{c,k} ∝ 子簇样本数 或 子簇困难度
```

如果你想少改一点，就按子簇样本数分配：

$$alloc_{c,k} = alloc_c \cdot \frac{n_{c,k}}{\sum_k n_{c,k}}$$

这样 Stage 2 的虚拟特征生成就能覆盖一个类的多个模式。

# 九、分类头怎么办？

这是一个关键问题。

你现在 Stage 2 的 `CosineClassifier` 是：

```
weight: [num_classes, proj_dim]
```

每类一个权重方向。

如果你要彻底多原型分类，可以改成：

```
weight: [num_classes, K, proj_dim]
```

logit 计算为：

$$logit_c = scale \cdot \max_k cos(z, w_{c,k})$$

也就是每个类别的 logit 由该类最匹配的原型权重决定。

这叫 **multi-prototype cosine classifier**。

代码逻辑类似：

```Python
# z: [B, D]
# weight: [C, K, D]

z = F.normalize(z, dim=-1)
w = F.normalize(weight, dim=-1)

```

## 1. 多原型 PCD

```Python
def multi_proto_pcd_loss(z, labels, prototypes, margin=0.85, proto_mask=None):
    """
    z: [B, D]
    labels: [B]
    prototypes: [C, K, D]
    proto_mask: [C, K], optional
    """
    z = F.normalize(z, dim=-1)
    prototypes = F.normalize(prototypes, dim=-1)

    class_protos = prototypes[labels]          # [B, K, D]
    sim = torch.sum(z.unsqueeze(1) * class_protos, dim=-1)  # [B, K]

    if proto_mask is not None:
        mask = proto_mask[labels]              # [B, K]
        sim = sim.masked_fill(~mask, -1e9)

    max_sim = sim.max(dim=1).values            # [B]
    loss = F.relu(margin - max_sim).mean()
    return loss
```

## 2. 多原型 EMA 更新

```Python
@torch.no_grad()
def update_multi_prototypes(prototypes, z, labels, momentum=0.96, proto_mask=None):
    """
    prototypes: [C, K, D]
    z: [B, D]
    labels: [B]
    """
    z = F.normalize(z, dim=-1)

    classes = labels.unique()

    for c in classes:
        c = int(c.item())
        z_c = z[labels == c]                    # [Nc, D]

        if z_c.numel() == 0:
            continue

        p_c = F.normalize(prototypes[c], dim=-1) # [K, D]
        sim = z_c @ p_c.T                        # [Nc, K]

        if proto_mask is not None:
            sim[:, ~proto_mask[c]] = -1e9

        assign = sim.argmax(dim=1)

        for k in assign.unique():
            k = int(k.item())
            z_ck = z_c[assign == k]

            if z_ck.shape[0] == 0:
                continue

            mu = z_ck.mean(dim=0)
            mu = F.normalize(mu, dim=0)

            prototypes[c, k] = momentum * prototypes[c, k] + (1 - momentum) * mu
            prototypes[c, k] = F.normalize(prototypes[c, k], dim=0)

    return prototypes
```

## 3. 多原型分类头

```Python
class MultiPrototypeCosineClassifier(nn.Module):
    def __init__(self, in_dim, num_classes, num_prototypes=3, scale=30.0):
        super().__init__()
        self.weight = nn.Parameter(torch.randn(num_classes, num_prototypes, in_dim))
        self.scale = scale

        x = F.normalize(x, dim=-1)
```

## 1. L2 和 cosine，在你这个任务里哪种更好？

对于你现在的医学长尾分类场景，我更建议：

$$\textbf{继续用 cosine，不要直接换成 L2}$$

原因有四个。

第一，你当前的投影空间本来就是归一化空间。  
在归一化后，L2 距离和 cosine 相似度其实高度等价：

$$\|z-p\|_2^2 = \|z\|^2+\|p\|^2-2z^\top p$$

如果：

$$\|z\|=1,\quad \|p\|=1$$

那么：

$$\|z-p\|_2^2 = 2-2\cos(z,p)$$

也就是说，在单位球面上，用 L2 和用 cosine 本质上都是在优化角度距离。区别主要是 margin 的尺度和梯度形式。

第二，你的 Stage 2 分类头是 `CosineClassifier`。文档里写得很清楚，Stage 2 使用的是 `CosineClassifier(in_dim, num_classes, scale)`，并且 Stage 2 是冻结 encoder 和 projector，只训练 classifier。

粘贴的 markdown (1)。md

  
如果 Stage 1 的原型学习改成非归一化 L2，而 Stage 2 的分类仍然用 cosine，会出现训练几何不一致：

```
Stage 1: 让特征按欧氏距离聚集
Stage 2: 用角度相似度分类
```

这会增加调参难度。

第三，医学图像长尾分类中，特征范数可能混入置信度、样本质量、类别频次等因素。  
L2 会同时受到“方向”和“模长”的影响；cosine 更关注方向。对于长尾类别，样本少、增强强、特征范数可能不稳定，用 cosine 通常更稳。

第四，你当前的 SPKD 也是基于 Gram 矩阵保持相似性结构，本质上也是在维护样本之间的相似度关系，而不是绝对欧氏坐标。

## 3. 原论文怎么让不同类原型相互不相似？和我们的方法有什么区别？

原论文主要靠 **triplet loss 中的 negative 项** 来拉开不同类。

$$L=L_{triplet}+\lambda L_{reg}$$ $$L_{triplet}=\max(0,m+\|a-p\|_2-\|a-n\|_2)$$

其中 $a,p,n$ 分别是 anchor、positive、negative feature vector。论文明确说，这个目标用于让同类距离更小、不同类距离更大。

间接实现类间分离。

$$L_{intra}=mean_{c,k\neq k'}\max(\cos(p_{c,k},p_{c,k'})-m_{intra},0)$$

总损失可以写成：

$$L=L_{cls}+\lambda_{pcd}L_{mpcd}+\lambda_{spkd}L_{spkd}+\lambda_{inter}L_{inter}+\lambda_{intra}L_{intra}$$

原型数建议不要让模型一开始“自动学”，而是先做一个简单可靠的版本：

```

medium 类：K = 2 或 3
tail 类：K = 1 或 2
```

或者更简单：

```

```


这比直接让模型自己学习 $K_c$ 稳得多，也更适合你的长尾医学分类任务。

## 2. 它的核心作用：让后面的原型知道前面已经覆盖了什么

```
p3 应该再补充还没覆盖的区域
```

## 3. 它对问题定义的意义是什么？

原文的问题定义是：

对于真实类别 $c_i$ 的查询样本 $q^{c_i}$，希望至少存在一个该类原型比所有其他类别原型更近：

$$\exists k\in\{1,...,N_{c_i}\}, \forall c_j:c_j\neq c_i  
\|p^{c_i}_k-q^{c_i}\| < \|p^{c_j}_{k'}-q^{c_i}\|$$

这个定义的关键是 **existential condition**，也就是“存在一个本类原型足够接近查询样本”。

Muzaddid和Beksi - 2026 - Person …

这和单原型完全不同。

单原型要求：

```
这个类的所有样本都要靠近同一个中心
```

多原型要求：

```
每个样本只需要靠近本类的某一个原型
```

所以问题从：

$$q \rightarrow p_c$$

变成了：

$$q \rightarrow \min_k d(q,p_{c,k})$$

也就是：

$$\hat{c}=\arg\min_c \min_k d(q,p_{c,k})$$

那么问题来了：

> 如果一个类有多个原型，这些原型应该怎么放，才能覆盖这个类的不同区域？

自回归结构的意义就在这里。它使得原型生成变成一个**条件集合生成问题**：

$$p_1 = f(\langle sos\rangle,M^c)$$ $$p_2 = f(\langle sos\rangle,p_1,M^c)$$ $$p_3 = f(\langle sos\rangle,p_1,p_2,M^c)$$

这样每个新原型都可以补充前面原型没有覆盖到的区域。

这比简单均值或 k-means 更可学习。

## Stage 2：冻结 encoder + projector，只训练 GCP 原型生成器

这一阶段专门学习：

$$GCP(\{z_i:y_i=c\}) \rightarrow \{p_{c,1},...,p_{c,K}\}$$

做法：

1. 预提取所有训练样本的 projected feature：

$$z_i = projector(encoder(x_i))$$

2. 每个 batch 不再是随机图片 batch，而是 **class episode batch**：

```
每次采样 B 个类别
每个类别采样 S 个样本作为 memory
GCP 为每个类别生成 K 个原型
再用 query 样本训练这些原型
```

类似：

```
class c:
    support set: z_c^support -> GCP -> p_c1, p_c2, p_c3
    query set: z_c^query -> 和所有类原型比较 -> prototype classification loss
```

训练损失可以用 cosine 版：

$$logit_{i,c}=s\cdot \max_k \cos(z_i,p_{c,k})$$ $$L_{proto-cls}=CE(logits,y)$$

再加类内防塌缩：

$$L_{intra}=mean\max(\cos(p_{c,k},p_{c,k'})-m_{intra},0)$$

再加类间分离：

$$L_{inter}=mean\max(\cos(p_{c,k},p_{c',k'})-m_{inter},0)$$

所以：

$$L_{gcp}=L_{proto-cls}+\lambda_{intra}L_{intra}+\lambda_{inter}L_{inter}$$

这比原文的 L2 triplet 更适合你，因为你当前 PCD 本来就是归一化 cosine margin，Stage 2 classifier 也是 CosineClassifier。

# 5. 你可以怎么实现 GCP module？

可以做一个轻量版，不用完全复现原文 6 层 decoder。

输入：

```Python
support_z: [B_cls, S, D]
```

其中：

```
B_cls = 一个 episode 中的类别数
S = 每类 support 样本数
D = proj_dim，例如 128
```

输出：

```Python
prototypes: [B_cls, K, D]
```

核心结构：

```Python
class AutoregressivePrototypeGenerator(nn.Module):
    def __init__(self, dim=128, num_layers=2, nhead=4, ff_dim=512, dropout=0.1):
        super().__init__()
        decoder_layer = nn.TransformerDecoderLayer(
            d_model=dim,
            nhead=nhead,
            dim_feedforward=ff_dim,
            dropout=dropout,
            batch_first=True,
            norm_first=True,
        )
        self.decoder = nn.TransformerDecoder(decoder_layer, num_layers=num_layers)
        self.sos = nn.Parameter(torch.randn(1, 1, dim))
        self.head = nn.Linear(dim, dim)

    def forward(self, memory, num_proto):
        """
        memory: [B_cls, S, D]
        """
        B, S, D = memory.shape
        generated = []
        tokens = self.sos.expand(B, 1, D)

        for _ in range(num_proto):
            out = self.decoder(tgt=tokens, memory=memory)
            p = self.head(out[:, -1, :])
            p = F.normalize(p, dim=-1)

            generated.append(p)
            tokens = torch.cat([tokens, p.unsqueeze(1)], dim=1)

        return torch.stack(generated, dim=1)  # [B_cls, K, D]
```

这就是原文自回归思想的最小实现。

# 6. GCP 训练时怎么构造 batch？

不要用普通 image batch，而是用 episode batch。

例如：

```
每个 iteration：
    采样 8 个类别
    每类采样 8 张图像
    其中 5 张作为 support
    3 张作为 query
```

得到：

```
support_z: [8, 5, 128]
query_z: [8*3, 128]
query_y: [8*3]
```

GCP 对每类 support 生成原型：

```
prototypes: [8, K, 128]
```

然后 query 和这些 prototypes 算 logits：

$$logit_{i,c}=s\cdot \max_k \cos(q_i,p_{c,k})$$

再做 CE。

这种训练方式和原文思想更接近：原文用同类多张 gallery features 作为 memory，生成 prototypes，然后用于 query 检索；你这里用 support set 生成 class prototypes，再分类 query。

### 方案 A：Stage 1 基本保持不变，先用单原型稳定空间

$$L_{stage1}=L_{cls}+\lambda_{pcd}L_{pcd}+\lambda_{spkd}L_{spkd}+\lambda_{sep}L_{proto\_sep}$$

## 3. Stage 2 的虚拟样本生成结果能不能指导原型生成？

可以，但要非常小心。

我的建议是：

> **第一版不要让虚拟样本直接训练 GCP 原型生成器。先让 GCP 只看真实样本。虚拟样本最多用于后续更新统计量或训练 classifier。**

原因是你当前 Stage 2 的虚拟样本本来就是根据类中心、共享协方差、当前 classifier 筛选出来的。它不是原始真实数据，而是模型假设下生成的特征。你的 Stage 2 当前每个 epoch 都会“生成 → 筛选 → 重算 mu/cov → 再生成”，并且不会跨 epoch 保存旧虚拟特征。

粘贴的 markdown (1)。md

这套闭环如果再反过来训练 GCP，会形成一个自举循环：

```
GCP 原型影响虚拟样本生成
虚拟样本又反过来更新 GCP 原型
更新后的 GCP 又影响下一轮虚拟样本
```

这可能有帮助，但也很容易放大错误。

特别是在长尾医学分类里，虚拟样本如果早期偏了，GCP 会把偏的虚拟样本当成类内分布的一部分，导致原型漂移。

所以第一版最好不要这样做。

## 4. 更清晰的 Stage 2 设计

我建议你把 Stage 2 拆成两个子阶段。

---

### Stage 2a：只用真实样本训练 / 生成多原型

输入：

$$z_i=projector(encoder(x_i))$$

冻结：

```
encoder frozen
projector frozen
```

训练 GCP：

$$P_c=GCP(\{z_i:y_i=c\})$$

其中：

$$P_c=\{p_{c,1},p_{c,2},...,p_{c,K}\}$$

训练损失建议用：

$$logit_{i,c}=s\cdot\max_k \cos(z_i,p_{c,k})$$ $$L_{gcp-cls}=CE(logits,y)$$

再加：

$$L_{inter}=mean\max(\cos(p_{c,k},p_{c',k'})-m_{inter},0)$$ $$L_{intra}=mean\max(\cos(p_{c,k},p_{c,k'})-m_{intra},0)$$

最终：

$$L_{gcp}=L_{gcp-cls}+\lambda_{inter}L_{inter}+\lambda_{intra}L_{intra}$$

这里 **不要引入虚拟样本**。

这样得到的是干净的：

```
prototype_memory_gcp: [C, K, D]
```

这一步对应原文 GCP 的核心做法：原文也是用已提取的特征作为 GCP 的 memory，再由 GCP 得到最终类别原型。

Muzaddid和Beksi - 2026 - Person …

---

### Stage 2b：用多原型指导虚拟样本生成

现在有了：

$$P_c=\{p_{c,1},...,p_{c,K}\}$$

再做虚拟样本生成。

原来你是每类一个中心：

$$\mu_c$$

现在改成每类多个子中心：

$$\mu_{c,k}$$

真实样本先分配给最近的原型：

$$assign(z_i)=\arg\max_k \cos(z_i,p_{c,k})$$

然后每个子簇计算：

$$\mu_{c,k}^{real}=mean(z_i), \quad assign(z_i)=k$$

融合 GCP 原型：

$$\tilde{\mu}_{c,k}=\lambda_\mu \mu_{c,k}^{real}+(1-\lambda_\mu)p_{c,k}$$

然后从每个子簇生成虚拟样本：

$$z_{virt}\sim \mathcal{N}(\tilde{\mu}_{c,k},\Sigma)$$

这时虚拟样本是被 GCP 指导的，而不是反过来指导 GCP。

这样逻辑很清楚：

```
真实样本 -> GCP 多原型
GCP 多原型 -> 多中心虚拟样本
虚拟样本 -> classifier
```

而不是：

```
真实样本 -> GCP
GCP -> 虚拟样本
虚拟样本 -> GCP
GCP 再变
```

后者更混乱。

这不是违背 GCP，而是更适配长尾数据。

## 9. 我建议你采用的最终版本

如果你要“完整改成严格符合 GCP”，我建议这样落地：

```
Stage 0:
    使用预训练 MedCLIP-ViT encoder
    可选训练一个轻量 projector
    提取并保存所有 train/val/test projected features

Stage 1:
    训练 autoregressive GCP generator
    输入每类 feature set
    输出每类 K 个 prototypes
    使用 max-over-prototypes 分类损失 + inter/intra regularization

Stage 2:
    使用 GCP prototypes
    做 multi-prototype classifier 或 multi-center virtual feature generation
```

如果你不想再出现“单原型”这个中间步骤，那么 Stage 1 里就不要再用原来的单原型 PCD：

$$L_{pcd}=max(m-\cos(z,p_y),0)$$

而是直接换成：

$$L_{gcp-cls}=CE(s\cdot\max_k\cos(z,p_{c,k}),y)$$

或者：

$$L_{mpcd}=max(0,m-\max_k\cos(z,p_{y,k}))$$

这样你的方法从一开始就是多原型，不再经过单原型过渡。

## 10. 一句话结论

**可以完整改成严格 GCP，而且你的预训练 ViT encoder 正好适合这样做。**

但严格 GCP 的关键不是“把 projector 变成 Transformer”，而是：

> 冻结或预先得到一个稳定 embedding space，然后用 Transformer Decoder 从同一类别的 feature set 中自回归生成多个互补的 class prototypes。

对你的项目来说，最清晰的新架构是：

```
预训练 ViT / projector 负责样本 embedding
GCP decoder 负责类级多原型生成
Stage 2 负责基于多原型做长尾校准和虚拟特征生成
```

这样既保留了原文 GCP 的核心机制，也避免了“先单原型、再多原型”的冗余叙事。

# 方案 A：Stage 1 直接多原型，不用自回归

这个方案回答的问题是：

> 在当前 TPCSD 的投影空间里，单原型约束是否过强？把每类一个原型改成多个原型，能不能缓解类内不成簇问题？

你当前 Stage 1 已经是：

```

```

也就是说，你只需要把 prototype memory 从 `[C, D]` 改成 `[C, K, D]`，再把 PCD 从“靠近唯一原型”改成“靠近本类最近原型”。当前 PCD 本来就是基于归一化特征和 cosine margin：

粘贴的 markdown (1)。md

$$L_{pcd}=mean_i\max(m-\cos(z_i,p_{y_i}),0)$$

多原型版就是：

$$L_{mpcd}=mean_i\max(0,m-\max_k\cos(z_i,p_{y_i,k}))$$

这个版本的优点是：

```
改动小
保留你原有 Stage 1 / Stage 2 逻辑
能直接验证多原型是否有收益
不用新增 Transformer Decoder
不用担心自回归训练不稳定
```

它的缺点是：

```
不是严格 GCP
多个原型之间没有生成顺序依赖
更像 online multi-centroid / multi-prototype memory
```

但对你的任务来说，这正好是最好的 baseline。如果连这个轻量多原型都没有收益，严格 GCP 大概率也不会稳定收益。

## Step 0：先做“类内多模态诊断”

在改模型前，先做一个分析脚本。对每个类别的特征 $z_i$ 或 $h_i$，计算这些指标：

### 指标 1：类内聚类倾向

对每个类分别做 k-means，比较：

$$K=1,2,3$$

看 silhouette score / Davies-Bouldin / Calinski-Harabasz。

如果某类 $K=2/3$ 明显优于 $K=1$，说明它可能真有多簇结构。

### 指标 2：类内方差与样本数关系

统计：

```
class_count[c]
intra_class_variance[c]
nearest_inter_class_distance[c]
```

如果“不成簇”主要出现在少样本类，并且这些类样本数很少，那么更可能是少样本估计不稳定，不是真多模态。

### 指标 3：多原型使用率

对每个类做 k-means $K=3$，看每个簇样本数：

```
[30, 28, 25]：比较像真实多簇
[80, 2, 1]：不适合多原型
[3, 2, 1]：样本太少，不可靠
```

这一步能帮你判断：长尾皮肤病数据到底有没有多原型必要。

## 5. 是否“不使用 projector”是好选择？

可以尝试，但我建议你不要把它设为唯一主线。

你的当前系统里，projector 的作用很关键：它把 backbone feature 映射到原型/对比空间，Stage 1 的 PCD、SPKD、prototype memory 都依赖它，Stage 2 也用冻结 projector 做特征抽取和虚拟特征建模。

粘贴的 markdown (1)。md

如果去掉 projector，优点是：

```
更接近原论文

避免 projector 单原型训练带来的类内压缩
```

但缺点是：

```

ViT 原始特征可能不适合高斯虚拟特征生成
Stage 2 要重构
维度高导致协方差估计更不稳定
```


这能判断到底是“projector 不好”，还是“当前单原型训练方式不好”。
